创建对象的几种模式以及优缺点

发表于 2018-03-20 | 分类于 js

一、使用Object构造函数或对象字面量

缺点：使用同一接口创建很多对象，产生大量的重复代码。

二、工厂模式
用函数来封装以特定接口创建对象的细节

优点：解决了创建多个相似对象的问题
缺点：没有解决对象识别的问题（即怎样知道一个对象的类型，当前都是Object类型）

三、构造函数模式
创建自定义构造函数，从而定义自定义对象类型的属性和方法

优点：相较于工厂模式，可以将实例标识为一种特定的类型 (Person)
缺点：1.每个方法都要在每个实例上重新创建一遍
2.sayName函数只是被某些对象调用，却写在全局作用域中

四、原型模式

优点：可以让所有实例共享原型的属性和方法
缺点：若包含引用类型的属性(friends),当在person1中修改friends时在person2中也有所体现

原型链：红宝书P148 图6-1

五、组合使用构造函数模式和原型模式
构造函数模式用于定义实例属性，原型模式用于定义共享的属性和方法。

该种模式是目前在ECMAScript中使用最广泛、认同度最高的一种创建自定义类型的方法。

六、动态原型模型
通过检查某个应该存在的方法是否有效，来决定是都需要初始化原型。

优点：if语句检查的可以是初始化之后应该存在的任何属性或方法，不必用一大堆if语句检查每个属性和每个方法。

七、寄生构造函数模式

返回的对象与构造函数或者构造函数的原型属性之间没有关系，构造函数返回的对象与在构造函数外部创建的对象没有什么不同。
不能依赖instanceof操作符来确定对象类型。

八、稳妥构造函数模式
稳妥对象：指没有公共属性，而且其方法也不引用this的对象。

优点：安全，除了使用sayName方法之外，没有其他办法访问name的值。

九、检测对象关系的相关方法

  1. isPrototypeOf()方法：
    console.log(Person.prototype.isPrototypeOf(person1)); //true

  2. Object.getPrototypeOf()方法：
    console.log(Object.getPrototypeOf(person1) == Person.prototype); //true
    console.log(Object.getPrototypeOf(person1).name); //“Nicholas”

  3. hasOwnProperty()方法：检测一个属性是存在于实例中，还是存在于原型中。
    console.log(person1.hasOwnProperty(“name”)); //false

  4. 确定属性是在对象中，还是在原型中
    function hasPrototypeProperty(object,name) {
    return !object.hasOwnProperty(name) && (name in object);
    }
