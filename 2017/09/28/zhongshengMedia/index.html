发表于 2017-09-28 | 分类于 项目总结

一、前期知识储备

第一次做移动端的页面，一开始整个人有点懵，不知道从何下手，靠着和伙伴讨论以及查阅资料、学习了慕课视频，才比较有方向。比较有价值的参考资料：
Hello,移动WEB
一篇真正教会你开发移动端页面的文章
上面的视频和文章都讲得很清楚了，不需要再多说什么，下面就只记一些关键点吧。

1.像素

设备像素：固定
CSS像素：双指缩放页面时，实际上是在放大或缩小CSS像素

2.移动端的三个视口

视口是html的父元素，初始包含块
在PC端 视口宽度=浏览器窗口宽度
布局视口
document.documentElement.clientWidth
document.documentElement.clientHeight
视觉视口
理想视口
CSS是根据布局视口计算的，故最终起作用的还是布局视口

3.meta标签

设置meta可告诉手机浏览器把布局视口，代码如下：

width：设置布局视口的宽
init-scale：设置页面的初始缩放程度
minimum-scale：设置页面最小缩放程度
maximum-scale：设置页面最大缩放程度
user-scale：是否允许用户缩放页面

2.关于页面布局

采用flex布局，需要注意的一点是不要随意添加padding、margin或者设置text-indent，因为有时候添加了上述样式，在iphone5s显示时文字可能会被挤换行，这个要视具体情况而定。

3.关于是否使用框架的问题

因此次项目是分为移动端和PC端的，故没有使用Bootstrap框架；项目中的样式比较简单，不需要使用UI框架，如果使用JQuery Mobile会有些笨重。

4.关于点击穿透事件

视频中也有提到几种点击穿透事件的解决办法，在本项目中使用的是fastclick.js
免佣租房搜索房源的时候，点击或者更准确点说触摸到遮罩层或者选项就应该把整个遮罩层和选项区域收起。
在一开始实现该功能的时候，我设置的是一touchstart就收起，这样一来就出现了看似点击穿透的bug。但其实此处已经引入了fastclick.js，不是点击穿透，而是一toucustart便收起了整块选择区和遮罩层，导致在touchmove的过程中点击到后面的房源，直接跳转到房源详情页面。
解决办法：把touchstart改成touchmove

7.关于三大系统的字体设置问题

参考自：移动web页面使用字体的思考-白树-博客园
+各个手机系统有自己的默认字体，且都不支持微软雅黑
+如无特殊需求，手机端无需定义中文字体，使用系统默认
+英文字体和数字字体可使用Helvetica，三种系统都支持

8.关于部分页面设置了width:100%仍能左右滑动的问题

这个bug纯属粗心了，此处是因为给input加了一个margin-left，而宽度又设置了100%，只需要把width:100%去掉即可。
